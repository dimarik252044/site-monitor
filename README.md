# Мониторинг сайта контракт-бюро.рф

Каждые 30 минут GitHub Actions открывает сайт в headless-браузере и проверяет:

1. Сайт открывается (HTTP 200, не белый экран)
2. Основной скрипт сайта выполнился, CDN-библиотеки загрузились
3. Кнопки не перекрыты невидимыми слоями
4. Форма «Получить консультацию» открывается
5. Поля заполняются, маска телефона работает
6. Заявка отправляется и данные в ней корректны
7. Посетитель видит окно «Заявка принята»
8. Нет ошибок JavaScript

Тестовая заявка **не доходит до менеджеров**: запрос перехватывается
внутри браузера и подменяется фейковым ответом Telegram.

При проблеме бот присылает алерт со скриншотом в Telegram.

## Запуск вручную

GitHub → Actions → «Проверка сайта» → Run workflow.

Локально: `TG_TOKEN=... ALERT_CHAT_ID=... npm run check`

## Секреты (Settings → Secrets and variables → Actions)

- `TG_TOKEN` — токен бота для алертов
- `ALERT_CHAT_ID` — chat_id получателя алертов

## Важно

GitHub отключает расписание, если в репозитории 60 дней нет активности —
раз в пару месяцев достаточно любого коммита или ручного запуска workflow.
